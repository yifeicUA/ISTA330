In the previous practicum, you added a PostgreSQL database to your imagequiz-backend API. Currently your imagequiz frontend reads its data(flowers and quizzes) from the data harcoded in the frontend. In this practicum you will modify your imagequiz frontend so that it will get its data(flowers and quizzes) from the imagequiz-backend API.
  • First you need to add one more table(to store scores) to the database in the backend. Table name should be 'score'. The table should have the following columns: id, customer_id, quiz_id, score, date (customer_id and quiz_id are foregin keys refereing customer and quiz tables respectively).
  • Then modify your '/score' method in your imagequiz-backend API so that it saves the score into the table score you created in the previous step.
  • Then delete the follwoing files from your front-end: customers.js, data.js, flowers.js and scores.js.
  • Modify your front-end so that it reads flowers and quizzes from your imagequiz-backend API. Refer to the lectures 'Fetch API' on d2l to laern how to use the fetch API to connect your front-end to your imagequiz-backend API.
  • Modify your front-end so that it sends the coustomer registeration information to the imagequiz-backend API to be saved.
  • Modify your front-end so that it uses the '/login' method in imagequiz-backend API to authenticate the user.
  • Modify your front-end so that it sends the score to the imagequiz-backend API to be saved.
